## What Is a Work Breakdown Structure (WBS)?

A **Work Breakdown Structure (WBS)** is a hierarchical tool that breaks down the **milestones** and **tasks** of a project into manageable components, organized in the order they need to be completed.
### Benefits of Using a WBS:

- Makes large projects feel **less intimidating**    
- Clarifies **step-by-step tasks** required to reach milestones    
- Lays out a clear path from **project start to finish**    

---

## Example: WBS for Project Plant Pals – Website Launch

### WBS Tree Diagram (Simplified)

```yaml
Project: Plant Pals Website Launch
│
├── Milestone: Design Approval
│   ├── Task: Mock up website designs
│   └── Task: Collect feedback
│
├── Milestone: Site Development
│   ├── Task: Code landing page
│   └── Task: Code contact page
│
└── Milestone: Implement Feedback
    └── Task: Revise design based on testing
```

> 🎯 The WBS is used for planning and organizing, not typically included in official project plans. Instead, use a spreadsheet or project management tool for execution.

---
## What Comes After WBS?

Once tasks are identified:
1. **Organize them in a spreadsheet or project tool**    
2. **Assign owners to each task**    
3. **Add clear due dates and descriptions* 

---
## How to Assign Tasks

Tasks are typically assigned based on:
- **Project roles**    
- **Expertise**    
- **Workload**    
- **Availability**    
### Example Task Assignments

|Task|Role|Assignee|
|---|---|---|
|Mock up designs|Web Designer|Taylor|
|Provide feedback|PM (You)|You|
|Code website|Web Developer|Jordan|

---
## Best Practices for Task Assignment

✅ **Start task names with a verb**

> e.g., "Mock up homepage" instead of just "homepage"

✅ **Add a due date and assignee**  
✅ **Include detailed descriptions**  
✅ **Use tools like Asana for clarity and collaboration**

---
## Why Assigning Tasks Matters

### Obvious Benefits:

- Frees you to **focus on project management**    
- Ensures **accountability** and timely execution    
### Less Obvious Benefits:

- Encourages **ownership and responsibility**    
- Promotes **personal growth** and delegation skills    
- Keeps team members **motivated and engaged**    
- Builds **team rapport** through shared support    

---
## Recap

- A **WBS** helps you break down project milestones into tasks.    
- Tasks can be **organized, assigned, and tracked** using software tools.    
- Assigning tasks with clarity fosters both **individual ownership** and **team accountability**.
