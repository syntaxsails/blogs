## Capacity Planning

**Capacity** refers to the amount of work people or resources assigned to a project can reasonably complete within a set period. It's vital to consider each person's capacity when assigning work, as individuals have limited capacity due to meetings, urgent tasks, and other daily work elements.

**Capacity planning** is the act of allocating people and resources to project tasks and determining if the necessary resources are available to complete the work on time. This process might reveal a need for more resources to meet project timelines.

**Example (Plant Pals Project)**: If 100 plant deliveries are needed over five days, and one driver averages four deliveries in an eight-hour workday, then at least five drivers are required to meet the deadline.

## Critical Path

To prioritize team members' time and maximize capacity, project managers can plot the

**critical path** of their project timeline.

The

**critical path** is a list of project milestones that must be achieved to meet the project goal on schedule, along with the mandatory tasks contributing to each milestone's completion. Tasks not on the critical path are considered less crucial for the project's overall success. Failure to complete critical path tasks on time can lead to project delays.

### Identifying the Critical Path

To determine the critical path:

1. **List all tasks and milestones**: Refer to your Work Breakdown Structure (WBS) for a detailed overview of every project task and how they feed into milestones.    
2. **Identify dependencies**: Determine which tasks cannot begin until another task is complete.    
3. **Estimate and map tasks**: Work with your team to make time estimates for each task and map each task from start to finish.    
4. **Longest path**: The longest sequence of dependent tasks is your critical path.    

### Factors Impacting Capacity and Critical Path Planning

Several factors influence capacity and critical path planning:

- **Parallel Tasks**: Tasks that can happen at the same time as other tasks. Identifying these creates efficiencies in the project schedule.    
    - **Example (Plant Pals Project)**: Hiring delivery drivers and developing a new website can be parallel tasks as they are independent and can be completed by different team members.  
- **Sequential Tasks**: Tasks that must happen in a specific order. Identifying these helps prioritize tasks early in the project.    
    - **Example (Plant Pals Project)**: Budget approval must occur before hiring a vendor.        
- **Fixed Start Date**: The specific date on which work must begin on a task to achieve the project goal. This helps ensure the right number of people are available.    
    - **Example**: If 100 plants must be delivered on a specific date, the task of picking them up has a fixed start date of one day prior.        
- **Earliest Start Date**: The earliest date a task can begin. Identifying this sets accurate expectations for when vendors and team members will be operational.    
    - **Example**: A new vendor cannot start until contracts are signed and a purchase order is approved, which might take three weeks. Thus, their earliest start date is three weeks from the kickoff meeting.        
- **Float (Slack)**: The amount of time a task can be delayed without impacting the project schedule or outcome.    
    - Tasks on the critical path should have        
        **zero float**, meaning no room for delays.        
    - Tasks with float are not part of the critical path.        
    - **Example**: The shipment of plants to a priority customer with a specific delivery date has zero float.

# Creating a Critical Path

The **critical path** serves as a framework, guiding the project manager on the project's current status, direction, and estimated completion time. It identifies essential tasks and their durations, highlighting those that, if delayed, will negatively impact the project completion date. It also helps define necessary resources, project baselines, and scheduling flexibility.

Why the Critical Path is Critical

- Determines essential tasks and their durations required to meet the end goal.
    
- Provides a quick reference for critical tasks that will impact the project completion date if delayed.
    
- Helps define necessary resources, project baselines, and scheduling flexibility.
    

## How to Create a Critical Path

While each project differs, these general steps apply to most projects:

### Step 1: Capture All Tasks

Begin by listing all tasks necessary for project completion, utilizing planning documents like the Work Breakdown Structure (WBS). Focus on essential "need-to-do" tasks, rather than "nice-to-do" tasks.

**Example Critical Tasks for House Structure:**

- A) Excavation    
- B) Foundation    
- C) Framing    
- D) Roof    
- E) Plumbing    
- F) Heating, Ventilation, and Air Conditioning (HVAC)    
- G) Electrical    
- H) Insulation    
- I) Drywall + Paint    
- J) Flooring    

### Step 2: Set Dependencies

Arrange critical tasks in their completion order by identifying

**dependencies**. A dependency means one task must be completed before another can start (e.g., framing must precede painting).

To determine dependencies, ask:

- Which task needs to take place before this task?    
- Which task can be finished at the same time as this task?    
- Which task needs to happen right after this task?    

**Example Task Dependencies:** 

| Task               | Dependency | 
| :-------------- | :----------------------------------- | 
| A) Excavation      | | 
| B) Foundation      | A) Excavation | 
| C) Framing         | B) Foundation | 
| D) Roof            | C) Framing | 
| E) Plumbing        | C) Framing |
| F) HVAC            | C) Framing |
| G) Electrical      | C) Framing | 
| H) Insulation      | E) Plumbing, F) HVAC, G) Electrical | 
| I) Drywall + Paint | H) Insulation | 
| J) Flooring        | I) Drywall + Paint |
  
### Step 3: Create a Network Diagram

Visualize the critical path using a network diagram, which sequences tasks based on dependencies. These diagrams show the path of work from start to finish, illustrating parallel (e.g., HVAC and plumbing) and sequential (e.g., plumbing then insulation) tasks. They also distinguish between essential and non-essential tasks.

### Step 4: Make Time Estimates

Consult stakeholders to obtain accurate time estimates for each task. Inaccurate estimates can alter the critical path's length. These estimates can be reviewed and updated as needed throughout the project.

**Example Task Durations:** 

| Task | Duration | Dependency | 
| :-------------- | :-------- | :----------------------------------- | 
| A) Excavation | 1 Day | | 
| B) Foundation | 3 Days | A) Excavation | 
| C) Framing | 15 Days | B) Foundation | 
| D) Roof | 3 Days | C) Framing | 
| E) Plumbing | 4 Days | C) Framing |
| F) HVAC | 3 Days | C) Framing | 
| G) Electrical | 3 Days | C) Framing |
| H) Insulation | 2 Days | E) Plumbing, F) HVAC, G) Electrical | 
| I) Drywall + Paint | 15 Days | H) Insulation |
| J) Flooring | 7 Days | I) Drywall + Paint |

### Step 5: Find the Critical Path

Add the estimated durations to your network diagram. Calculate the longest possible path by summing the durations of only the "essential" tasks that impact the project's finish date if left uncompleted (e.g., landscaping and driveway pavement are non-essential for house structure completion).

Two common approaches for calculating the critical path are:

- **Forward Pass**: Start at the beginning of your task list and add up the durations of critical path tasks to the project's end.    
- **Backward Pass**: Start with the final task or milestone and move backward through your schedule to determine the shortest path to completion, especially useful for hard deadlines to identify tasks that can be cut or completed later.

These techniques help identify earliest and latest start dates and

**slack** (also known as **float** ), which is the amount of time a task can be delayed without delaying the project. Tasks on the critical path should have zero float.
