## What is Computer Architecture?

- **Computer architecture** refers to the design, structure, and functionality of computer systems.
- Like buildings, different computers are designed based on their specific use cases.
- It's the set of **rules and methods** that describe how hardware and software components are organized and interact.


---

## Components of Computer Architecture

Computer architecture can be broadly divided into:

- **Hardware Design**
- **Software Design**

Each has its own layers and subcomponents.

---

## Hardware Components

Hardware includes all **physical components** of a computer system:

- **CPU (Central Processing Unit)**
- **Memory (RAM)**
- **Storage Devices** (Hard drives, SSDs)
- **Input/Output Devices** (I/O adapters, network cards)
- **GPU (Graphics Processing Unit)**

These are the tangible elements you can physically interact with.

---

## Software Components

Software refers to the **intangible programs and instructions** that control the hardware.

### 1. **System Software**

- Includes operating systems and low-level utilities.
- Manages hardware resources and provides services to application software.
- Most application software runs on top of the operating system.

### 2. **Application Software**

- User-facing programs designed for specific tasks.
- Examples: Word processors, web browsers, games.
- Typically rely on system software to access hardware.

### 3. **Firmware**

- Specialized software stored in **read-only memory (ROM)**.
- Provides low-level control for hardware components.
- Usually pre-installed and updated only when necessary.

---

## The Role of Compilers

- Code written in high-level languages (e.g., Java, C++) must be **compiled** into machine-readable code.
- The **compiler** translates source code into **object code**.
- This is a **one-way process** (you can't reverse object code into source code easily).
- Analogy: Compiling is like baking a cake—you can’t turn the cake back into ingredients.

---

## Instruction Set Architecture (ISA)

- The **Instruction Set Architecture (ISA)** defines how software communicates with hardware.
- It includes all operations a CPU can perform:
    - Arithmetic operations (e.g., add, compare
    - Data movement (e.g., store, load)
    - Control operations (e.g., branch, jump)
- Acts as a bridge between hardware and software.

### Examples:

- A phone, calculator, and supercomputer each have:
    - Different architectures
    - Different instruction sets
    - Unique compilers suited to their architecture

---

## Physical Design

- Encompasses how the logic is **physically implemented** on silicon chips.
- Includes:
    - **Digital Design** (logic gates, control circuits)
    - **Circuit Implementation**
    - **Physical Layout** (chip fabrication)

This field goes deeper into **electrical engineering and hardware fabrication**.

---

## Summary

- Computer architecture spans multiple layers:
    1. **Applications** written by users
    2. **System software** that manages resources
    3. **Firmware** that initializes hardware
    4. **Instruction set** that defines CPU capabilities
    5. **Hardware** that executes the instructions
    6. **Physical implementation** that brings all logic into reality

Each layer builds upon the one below it, enabling modern computing.
