## Overview

- The term **mainframe** refers to a family of computers sharing a common architecture, not a single specific machine.
- This course focuses on the **core fundamentals of mainframe architecture**, especially the memory model.

---
## Memory in Mainframes

- Like all computers, mainframes require memory for applications and operating systems.
- Physical memory is limited, so **auxiliary storage** (also called **virtual memory**) is used to extend available memory.

### Key Concepts

- **Real Memory (Central Storage)**: Actual physical memory (RAM).
- **Auxiliary Storage (AUX Storage)**: Disk-based memory extension, stored on **DASD** (Direct-Access Storage Devices).
    - DASD can be hard drives or solid-state drives.
    - Slower than real memory but provides scalability.
- The term **storage** refers to both real memory and auxiliary storage due to their tight integration in the architecture.

---
## Virtual Memory and Paging

- Mainframe architecture presents a unified memory model to applications, giving the illusion of large or "unlimited" memory.
- In **z/OS**, the mainframe OS, users and applications each receive an **address space**.

### Address Space

- A **map of storage** made available to an application.
- Addressable up to **16 exabytes** (1 exabyte = 1,024 petabytes).

### Paging Mechanism

- Memory is divided into **4 KB pages**.
    - In real memory: a page occupies a **frame**.
    - In auxiliary storage: a page occupies a **slot**.
- **Paging**: Movement between frames and slots.
    - Ideally occurs minimally for performance.      
    - Excessive paging reduces system efficiency.    

### OS Role in Memory Management

- z/OS periodically checks memory usage to determine if pages can be migrated.   
- Paging decisions and memory states are managed via **system-maintained tables**. 
- A well-tuned system ensures minimal page-back events and maintains stability.

---

## z/Architecture

- The current mainframe architecture standard is called **z/Architecture**.
- It includes:
    - Hardware
    - Operating systems (e.g., z/OS) designed to leverage hardware capabilities

---

## Summary

- Mainframes manage memory through a combination of real and auxiliary storage, collectively referred to as **storage**.
- z/OS provides each application with a massive virtual memory space via address spaces.
- Efficient memory handling through paging and system-level optimizations makes the mainframe's approach unique and powerful.
