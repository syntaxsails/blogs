## Overview

- **Virtualization** on the mainframe allows a single physical machine to host multiple logical systems.
- The distinction between **physical** (hardware) and **logical** (virtualized components) is key. 
- This flexibility is a cornerstone of mainframe design.

---

## Logical Partitions (LPARs)

- An **LPAR (Logical Partition)** is a virtual system created from a subset of physical resources.
- Acts like a standalone computer: runs its own OS, has virtual display, can be independently booted and shut down.

### Creating LPARs

- Physical resources (processors, memory, I/O adapters) are pooled and allocated to form LPARs.
- Each LPAR is isolated but runs on the same physical mainframe.
- Mainframe may host multiple LPARs:
    - e.g., 16 LPARs with varying configurations (more CPUs, more memory, more I/O).   

### Benefits

- Full system flexibility and resource control.
- Physical mainframe remains unchanged, but logically it hosts multiple independent systems.

---
## Processor Types in Virtualization

Mainframes use several processor types, each optimized for specific tasks:

|Processor Type|Description|
|---|---|
|**CP (Central Processor)**|General-purpose processor for OS and applications.|
|**SAP (System Assist Processor)**|Handles I/O operations, offloading work from CP.|
|**IFL (Integrated Facility for Linux)**|Dedicated to running Linux; has cost-effective licensing.|
|**zIIP (z Integrated Information Processor)**|Offloads specific workloads (e.g., Java, XML), improving performance and reducing costs.|

---

## I/O Virtualization and Adapters

### Channel Path Identifiers (CHPIDs)

- **CHPID**: Identifies physical I/O paths.
- **PCHID**: Physical Channel ID (the hardware port).
- CHPIDs are mapped to logical subsystems and are critical for LPAR communication.

### CHPID Configuration Options

- **Dedicated**: Fully assigned to a single LPAR.
- **Shared**: Accessible by multiple LPARs.
- **Unshared/Reconfigurable**: Switchable between LPARs but used by only one at a time.
- **Spanned**: Accessible from multiple logical channel subsystems simultaneously.

### Subsystems

- A **subsystem** links LPARs to I/O devices and other LPARs.
- Can be configured as a single large subsystem or divided into logical subsystems.

---

## z/VM: Virtualization Operating System

- **z/VM (Virtual Machine)** is a **type 1 hypervisor**—runs directly on hardware.
- Capable of hosting **hundreds to thousands of virtual machines (VMs)**.
- VMs share z/VM's allocated physical resources.
- Supports nesting:
    - z/VM can host any mainframe-compatible OS, including another instance of z/VM.

---

## Summary

- **Virtualization** transforms a powerful physical mainframe into multiple independent logical systems.
- **LPARs** act as full-fledged computers, created by assigning subsets of hardware.
- Multiple **processor types** optimize workload distribution and performance.
- **z/VM** enables high-density virtualization with advanced resource sharing.
