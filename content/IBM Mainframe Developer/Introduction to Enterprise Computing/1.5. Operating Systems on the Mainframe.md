## Introduction

Once an LPAR (Logical Partition) is created on a mainframe, an operating system (OS) must be installed to make it functional. IBM mainframes support a variety of operating systems, each suited for different purposes.

---
## IBM Mainframe Operating Systems

### 🟦 `z/OS`

- **IBM’s flagship mainframe OS**, built for high performance, scalability, and availability.    
- Descendant of OS/360 (1964).    
- Designed for running **mixed workloads** with **high Quality of Service (QoS)**.    
- Built for **z/Architecture** (64-bit).    
- Will be discussed in more detail later.    

---
### 🟦 `z/TPF` (Transaction Processing Facility)

- Built for **real-time, high-volume transaction processing**.    
- Commonly used in:    
    - **Credit card processing**        
    - **Airline reservation systems**        
    - **Hotels and car rental systems**        
- Strong ties to **travel and logistics** industries.    

---
### 🟦 `z/VM` (Virtual Machine)

- **Type 1 Hypervisor** (runs directly on hardware).    
- Hosts guest operating systems, offering full resource access.    
- Efficient for **testing, development**, and **running multiple OS instances** without using more LPARs.    
- Any OS that runs on the mainframe can be run under `z/VM`, including another instance of `z/VM`.    

---
### 🟦 `z/VSE` (Virtual Storage Extended)

- A **lighter alternative to z/OS**.    
- Suitable for **small to medium-sized businesses**.    
- Supports both **batch and transaction processing**.    
- Requires fewer personnel to manage.    

---
### 🐧 **Linux on the Mainframe**

- Fully supported and widely used.    
- Offers the same experience as Linux on other platforms.    
- Scalable and well-integrated.    
- Once controversial, it is now a **mature and popular option** for enterprise workloads.    

---
### 🟦 KVM (Kernel-based Virtual Machine)

- Open-source **Type 1 Hypervisor**.    
- Integrates with other KVM environments.    
- Ideal for **distributed and scalable workloads**:    
    - **Horizontal scaling**: Add more instances.        
    - **Vertical scaling**: Add capacity on demand.        

---

# z/OS: The Mainframe's Premier Operating System

## Key Characteristics

- Designed specifically for **z hardware**.    
- Delivers:    
    - High throughput        
    - High availability        
    - Robust security        
    - Mixed workload management        

---
## z/OS Components

### 🔧 Component Types

- **Base Elements**: Installed and configured by default.    
- **Features**: Optional add-ons that enhance functionality.    

---
### Core z/OS Components

- **Transaction Manager**: Manages transactions across systems.    
- **Databases**: Organize and store data for quick access.    
- **Batch Jobs**:    
    - Handle large data processing jobs outside peak hours.        
    - Written in **JCL (Job Control Language)**.        
    - Submitted to **JES (Job Entry Subsystem)**.        
    - Output viewed using **SDSF (System Display and Search Facility)**.        

---
### Security and Encryption

- **Security Manager**: Ensures only authorized access.    
- **Crypto Services**: Provide encryption and decryption.    

---
### Interfaces

- **TSO (Time Sharing Option)**:    
    - Supports thousands of users sharing one system interactively.        
- **USS (Unix System Services)**:    
    - POSIX-compliant UNIX environment within z/OS.        
    - Run shell scripts, utilities, and even submit z/OS jobs from the shell.        

---

### System and Resource Management

- **SMP/E (System Modification Program/Extended)**:    
    - Manages software installs, upgrades, and dependencies.        
- **RMF (Resource Measurement Facility)**:    
    - Tracks performance metrics and usage trends.        
- **DFSMS (Data Facility Storage Management Subsystem)**:    
    - Handles storage, backup, migration, and I/O management.        
- **WLM (Workload Manager)**:    
    - Allocates system resources based on predefined goals.        
    - Enables priority-based execution across LPARs or workloads.        

---

### Networking and Logging

- **SNA TCP/IP**:    
    - Networking protocol suite for all system communications.        
- **SMF (System Management Facility)**:    
    - Records system events in a standardized format.        
    - Essential for diagnostics, auditing, and tracing.        

---
## Summary

The IBM mainframe supports a diverse range of operating systems tailored for various enterprise needs. Among these, **z/OS** stands out for its robustness, scalability, and capability to manage complex and critical workloads. Understanding its components and supporting systems is essential for efficient mainframe operation.
